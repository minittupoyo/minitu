# デザインガイドライン

このプロジェクトのデザイン規則と、一貫性を保つためのガイドラインをまとめます。

## 1. デザインコンセプト

**「ミニマリズムと可読性の追求」**
装飾を最小限に抑え、余白とタイポグラフィを活かしたクリーンな設計を目指します。コンテンツが主役であり、ユーザーが情報に集中できる環境を提供します。

## 2. カラーパレット

Tailwind CSS のデフォルトカラーをベースに、落ち着いた配色を採用しています。

| 用途                      | クラス名           | 色味 (目安) |
| :------------------------ | :----------------- | :---------- |
| **背景色**                | `bg-white`         | #FFFFFF     |
| **基本テキスト**          | `text-slate-900`   | #0F172A     |
| **補助テキスト**          | `text-slate-500`   | #64748B     |
| **境界線 (強)**           | `border-slate-200` | #E2E8F0     |
| **境界線 (弱)**           | `border-slate-100` | #F1F5F9     |
| **アクセント (ホバー等)** | `text-slate-700`   | #334155     |

## 3. タイポグラフィ

可読性を重視し、以下のフォントスタックを基本としています。

- **フォント**: `Inter Variable` (英数), `Noto Sans JP Variable` (日本語)
- **文字色**: 原則として `slate-900` を使用し、コントラストを確保します。
- **レンダリング**: `antialiased` を適用し、滑らかな表示を実現します。

## 4. レイアウト規則

### コンテナ

- ページ全体の最大幅は `max-w-7xl` (1280px) とします。
- 左右には `px-6` の余白を設け、モバイル環境でもコンテンツが端に張り付かないようにします。

### ヘッダー

- `fixed top-0` で上部に固定されます。
- `bg-white/70` と `backdrop-blur-sm` を組み合わせ、スクロール時も背後が薄く透けるモダンな質感を維持します。
- 下部には `border-b border-slate-200` を配置し、コンテンツとの境界を明確にします。

## 5. コンポーネント設計

### Blog Card

- **形状**: `rounded-2xl` (16px) の大きな角丸。
- **枠線**: 控えめな `border-slate-100`。
- **インタラクション**:
    - 全体がリンクとして機能します。
    - ホバー時にはチェブロンアイコン（右矢印）がアニメーションしながら出現し、リンクであることを示唆します。
    - `transition-all duration-300` を用いた滑らかな変化。

## 6. コンテンツ作成ルール（Markdown）

ブログ記事の入稿時は、以下のルールを厳守してください。

- **見出し**: 本文中の見出しは `##` (h2) から開始すること。`#` (h1) は記事タイトルとしてシステム側で出力されるため、本文中では使用しません。
- **フロントマター**: 以下の項目を必ず含めること。
    - `title`: 記事のタイトル
    - `description`: 概要文（SNSなどのメタデータにも使用）
    - `emoji`: 記事を象徴する絵文字（1つ）
    - `date`: 公開日 (YYYY-MM-DD 形式)
    - `tags`: 関連するタグの配列
    - `draft`: 公開準備中は `true`、公開時は `false`
- **画像**: 必要に応じて `generate_image` 等で作成した高品質な画像を使用し、プレースホルダーは避けること。
