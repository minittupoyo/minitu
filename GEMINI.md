# GEMINI.md

このファイルは、AI エージェントがこのプロジェクトを理解し、効率的に作業を遂行するためのガイドラインです。

## 1. プロジェクト概要

- **名称**: minitu (みにっつ)
- **種類**: Astro ベースの個人ブログ
- **目的**: 技術的な知見の共有と実験的な UI 実装の場
- **主要な技術スタック**:
  - フレームワーク: Astro 6
  - スタイリング: Tailwind CSS 4
  - パッケージマネージャ: **Bun**
  - 言語: TypeScript

## 2. エージェントへの指示（最重要）

- **言語**: ユーザーへの提案、解説、コード内のコメント、およびプランニングなどの全ての対話は原則として**日本語**で行うこと。
- **実行環境**: パッケージのインストールやスクリプトの実行が必要な場合は、`npm` や `yarn` ではなく `bun` コマンドを使用すること。
- **コンテンツ規則**: ブログ記事を作成する際は、見出しを `##` (h2) から開始すること。

## 3. ディレクトリ構造

- `/src/pages`: ページファイル（`.astro`, `.md`）。ルーティングを定義。
- `/src/layouts/Layout.astro`: サイト全体の共通レイアウト。
- `/src/components`: 再利用可能な UI コンポーネント。
- `/src/styles/global.css`: Tailwind v4 のメインスタイル定義。
- `/content/blog`: ブログ投稿データ（Markdown）。Content Collections で管理。
- `/template`: 記事作成用のテンプレートファイル。

## 4. コーディング規約

- **パスエイリアス**: `src/` ディレクトリは `@/` エイリアスで参照すること。
- **コンポーネント**: Astro コンポーネントは機能ごとに分割し、再利用性を高めること。
- **スタイル**: 原則として Tailwind CSS のユーティリティクラスを使用し、コンポーネントごとに `DESIGN.md` の規則に従うこと。
- **フォーマット**: Prettier の設定に従い、インデントやクオートのスタイルを一貫させること。

## 5. 記事作成の推奨事項

- **トピック**: 技術的な内容を優先的に選択する。
- **メタデータ**: フロントマターには絵文字、タイトル、説明、正確な日付、関連するタグ（配列形式）を必ず含める。
- **トーン**: 丁寧かつ専門的なトーンを維持する。
